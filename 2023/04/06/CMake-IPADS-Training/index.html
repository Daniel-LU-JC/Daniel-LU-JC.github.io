first time to learn something about CMake and Makefile

Start with Makefile

Makefile defines the rules as to how to compile the project using a tool provided by GNU — make. CMake is built on top of Makefile-like systems which will be generated later based on CMake files.

Learn Makefile by Examples

From: richardchien/modern-cmake-by-example

branch 0: hello world

branch 1: “elegant” hello world

branch 2: ask for answer

Basic grammar rules of Makefile are demonstrated above and some of the variables can be passed from command line at the terminal.

Learn Modern CMake by Examples

branch 4: switch to CMake

How to use CMake to generate and run executable files?

branch 5: split library

‘add_library’ and ‘target_link_libraries’ are advisable to use if a certain library needs to be reused for many times in different executable files.

branch 6: sub-directory

Then, in the subdirectory answer, ‘libanswer’ library should be created with rules defined by CMake.

branch 7: use libcurl

CMakeLists.txt in the root directory does not need to change.

branch 8: link libs in the same root directory

As for the following three sub-directories/modules, it is important to know how they are dependent on each other, therefore the libraries can be linked correctly.

Branch eight gives us a great demonstration as to how to manage a project with hierarchical structure. In the following post, we are going to explore more about Modern CMake. To be continued…
